ArrayList

An ArrayList stores the sequence in a heap allocated array elements. The object also stores capacity and size.

  • Append: ensure capacity, write at elements[size], then increment size. Complexity: amortized O(1); O(n) in the occasional worst case when reallocation copies the existing elements.
  • Prepend: insert at index 0, shifting all current elements one position to the right before writing the new value. Complexity: O(n).
  • Insert: shift elements from the insertion point to the end one position to the right, write the new value, then increment size. Complexity: O(n) in the worst case; O(1) when inserting at the end and no reallocation is needed.
  • Get: direct access to elements[index]. Complexity: O(1).
  • Set: direct access to elements[index] followed by overwriting that cell. Complexity: O(1).
  • Delete: shift elements after the removed position one cell to the left and then decrement size. The old last physical cell is not cleared; it becomes outside the logical list. Complexity: O(n) in the worst case; O(1) when deleting the last element.
  • Clear: set size to 0. The allocated array and its old physical values are kept. Complexity: O(1).
  • Ensure capacity: when the array is full, realloc doubles the capacity and preserves stored elements by copying them into the new array. Complexity: O(n) for that resizing step.

Summary of asymptotic complexity

  • Append: amortized O(1), worst-case O(n).
  • Prepend: O(n).
  • Insert: worst-case O(n), best case O(1) at the end without resizing.
  • Get: O(1).
  • Set: O(1).
  • Delete: worst-case O(n), O(1) when deleting the last element.
  • Clear: O(1).
