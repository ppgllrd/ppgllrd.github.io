LinkedStack

A LinkedStack stores a sequence of elements using a Last-In-First-Out (LIFO) policy as nodes connected by references. Each node contains an element reference that points to the stored object, and a next reference that points to the next node, or to null when there is no next node.

The stack object keeps two pieces of information: top, a reference to the topmost node; and size, the number of elements currently stored. In a stack, elements are always inserted and removed from the top.

Operations and asymptotic complexity

  • Push: adds a new node at the top. Complexity: O(1), because the algorithm updates top directly.
  • Pop: removes the node at the top. Complexity: O(1), because the algorithm updates top.
  • Top: reads the element at the top without removing it. Complexity: O(1).

The visualization shows how links and temporary references change while each algorithm progresses, including how the Java Garbage Collector frees nodes that become unreachable.
