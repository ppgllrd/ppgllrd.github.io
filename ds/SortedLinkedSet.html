SortedLinkedSet

A SortedLinkedSet stores a sequence of unique elements in ascending order as nodes connected by references. Each node contains an element reference that points to the stored object, and a next reference that points to the next node.

The set object keeps two pieces of information: first, a reference to the front node; and size, the number of elements currently stored.

To implement operations, this structure uses a private Finder utility class. The Finder traverses the set searching for an element, and returns an object containing three fields: previous, a reference to the node immediately preceding the insertion/deletion point; current, a reference to the node at the insertion/deletion point; and found, a boolean indicating if the exact element was found.

Operations and asymptotic complexity

  • Insert: searches using Finder and adds a new node if it doesn't exist. Complexity: O(n).
  • Delete: searches using Finder and removes the node if it exists. Complexity: O(n).
  • Contains: runs Finder and returns the found field. Complexity: O(n).
  • Iterate: creates an iterator whose current reference visits each node in sorted order. Complexity: O(n) for visiting all elements.
  • Union: traverses the two sorted sets using elem1 and elem2. The smaller current element is appended to a temporary builder; when both current elements are equal, the value is appended once and both references advance. Complexity: O(n + m).
  • Intersection: traverses both sorted sets in parallel. If one current element is smaller, only that reference advances; equal elements are appended to the builder and both references advance. Complexity: O(n + m).
  • Difference: traverses this set and the other set in parallel. Elements from this set are appended only when they are not present in the other set; equal elements are skipped. Complexity: O(n + m).
