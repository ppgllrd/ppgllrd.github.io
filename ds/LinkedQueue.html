LinkedQueue

A LinkedQueue stores a sequence of elements using a First-In-First-Out (FIFO) policy as nodes connected by references. Each node contains an element reference that points to the stored object, and a next reference that points to the next node, or to null when there is no next node.

The queue object keeps three pieces of information: first, a reference to the front node; last, a reference to the rear node; and size, the number of elements currently stored. Keeping last allows the Enqueue operation to run without traversing from the beginning.

Operations and asymptotic complexity

  • Enqueue: adds a new node at the end (rear). Complexity: O(1), because the algorithm uses last.
  • Dequeue: removes the node at the beginning (front). Complexity: O(1), because the algorithm updates first.
  • First: reads the element at the beginning without removing it. Complexity: O(1).

The visualization shows how links and temporary references change while each algorithm progresses, including how the Java Garbage Collector frees nodes that become unreachable.
