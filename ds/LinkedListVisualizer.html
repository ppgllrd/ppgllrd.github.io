LinkedList

A LinkedList stores a sequence of values as nodes connected by pointers. Each node contains an element field with the stored value and a p_next field that points to the next node, or to NULL when there is no next node.

The list object keeps three pieces of information: p_first, a pointer to the first node; p_last, a pointer to the last node; and size, the number of nodes currently stored. Keeping p_last allows some operations at the end of the list to run without traversing from the beginning.

Operations and asymptotic complexity

  • Append: adds a new node at the end. Complexity: O(1), because the algorithm uses p_last.
  • Prepend: adds a new node at the beginning. Complexity: O(1), because the algorithm updates p_first.
  • Insert: adds a new node at a selected index. Complexity: O(n) in general, because the algorithm may need to traverse the list to find the insertion point. Inserting at the beginning behaves like Prepend; inserting at the end behaves like Append.
  • Get: reads the value at a selected index. Complexity: O(n) in general. Reading the last node can be O(1) by using p_last.
  • Set: replaces the value at a selected index. Complexity: O(n) in general. Updating the last node can be O(1) by using p_last.
  • Delete: removes the node at a selected index. Complexity: O(n), because the algorithm locates the node to delete and, when needed, the previous node. Deleting the first node updates p_first; deleting the last node also updates p_last.
