LinkedList

A LinkedList stores a sequence of elements as nodes connected by references. Each node contains an element reference that points to the stored object, and a next reference that points to the next node, or to null when there is no next node.

The list object keeps three pieces of information: first, a reference to the first node; last, a reference to the last node; and size, the number of elements currently stored. Keeping last allows some operations at the end of the list to run without traversing from the beginning.

Operations and asymptotic complexity

  • Append: adds a new node at the end. Complexity: O(1), because the algorithm uses last.
  • Prepend: adds a new node at the beginning. Complexity: O(1), because the algorithm updates first.
  • Insert: adds a new node at a selected index. Complexity: O(n) in general, because the algorithm may need to traverse the list to find the insertion point. Inserting at the beginning behaves like Prepend; inserting at the end behaves like Append.
  • Get: reads the element at a selected index. Complexity: O(n) in general. Reading the last node can be O(1) by using last.
  • Set: replaces the element at a selected index. Complexity: O(n) in general. Updating the last node can be O(1) by using last.
  • Delete: removes the node at a selected index. Complexity: O(n), because the algorithm locates the node to delete and, when needed, the previous node. Deleting the first node updates first; deleting the last node also updates last.

The visualization shows how links and temporary references change while each algorithm progresses.
